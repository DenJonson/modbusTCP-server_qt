Сервер, использующий протокол связи MODBUS-TCP

Данный проект реализует MODBUS-TCP сервер, запускающийся в локальной сети на порту 502.

Данный сервер рекомендуется использовать в связке с клиентом: https://github.com/DenJonson/modbusTCP-client_qt (на других не проверялся)

Сервер разработан на основе информации с сайта: https://ru.wikipedia.org/wiki/Modbus

Инструкция эксплуатации

  1. Запуск сервера

    При запуске сервера сверху окна будет отобажаться его статус, IP и используемый порт

  2. Управление сервером

    Задача сервера заключается в приеме запросов от клиента и формировании ответов на этот запрос. В поле "Данные сервера" можно изменять данные входов и адрес устройства вручную.

    В панелях сервера отображены данные входов/выходов, а также адреса, по которым можно получить к ним доступ.

  3. Состояние сервера

    В панели "Состояние сервера" можно увидеть процесс приема и обработки полученных запросов.

    Обрабатываемые ошибки:

    1. Неподдерживаемая функция

      При приеме от клиента запрос на выполнение функции, не входящей в диапазон 1-5, 0x0F и 0x10, будет возвращена ошибка с кодом 1

    2. Недопустимый адрес устройства

      При приеме от клиента запроса, предназначенного для устройства с Id отличным от значения в поле сервера "Адрес устройства", будет возвращена ошибка с кодом 2

    3. Недопустимое поле данных

      При приеме от клиента запроса на выполнение операции по адресу, отсутствующим в полях "Адрес 0x" сервера, будте возвращена ошибка с кодом 3. Причем, если запрос подразумевает действия с несколькими регистрами, даже если их не существует, но начальный адрес валидный, то ошибка выставлена не будет и действие будет произведено со всеми найденными ячейками.
